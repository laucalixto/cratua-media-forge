## Bugs encontrados

### B1. Profile `baseline` inválido para H.265
**Erro:** `-profile:v baseline` com `-c:v libx265` → exit code -22
**Causa:** O dropdown de profile tem opções fixas (baseline/main/high) que são de H.264. Ao selecionar H.265, o profile `baseline` é inválido.
**Correção:** Filtrar dinamicamente as opções de profile conforme o codec selecionado.

| Codec | Profiles válidos |
|---|---|
| H.264 | baseline, main, high |
| H.265 | main, main10, main12 |
| VP9, AV1, Copy | nenhum (ocultar dropdown) |

### B2. CRF + bitrate simultâneos conflitam
**Erro:** `-crf 30 -b:v 5000k` — ffmpeg não aceita bem ambos
**Causa:** Dragon trap só verifica se pelo menos um está setado, mas permite ambos
**Correção:** Se CRF > 0, ignorar bitrate (CRF tem prioridade). Mostrar aviso na UI.

### B3. Seta do Pixel Format & Threads invertida
**Erro:** ▶ abre o painel e ▼ fecha (deveria ser o contrário visualmente)
**Causa:** CSS `rotate(90deg)` no ▶ faz ele apontar para cima em algumas fontes
**Correção:** Trocar CSS por toggle explícito de texto: `▶` fechado ↔ `▼` aberto

## Validações adicionais

### V1. Codec + Container compatibilidade
- VP9 → força container WebM
- AV1 → sugere MP4/MKV
- MP3 codec → força container MP3

### V2. Profile dropdown dinâmico
- Ao trocar codec (#a-vcodec), atualizar opções do #a-profile
- Se profile atual é inválido para o novo codec, resetar para "auto"

### V3. Campos que precisam de validação
| Campo | Validação |
|---|---|
| Width/Height | min 2, max 7680, par (even) |
| CRF | 0-51, se 0 = lossless (avisar) |
| Video Bitrate | se > 0 e CRF > 0 = warning |
| Threads | 0 = auto (padrão correto) |
| Trim start/end | formato HH:MM:SS ou segundos |
