## 2. Presets Customizados (Salvar/Excluir)

### 2.1 Renomear "Legacy" → "Padrão"
**Arquivo:** `crates/mediaforge-core/src/preset.rs` linha 12
- `id: "legacy-bat"` → `id: "default"`
- `name: "Legacy (RenderConverter .bat)"` → `name: "Padrão (H.264 CRF 19)"`

### 2.2 UI de presets
**Arquivo:** `crates/mediaforge-tauri/ui/index.html`
- Adicionar seção "Presets" abaixo do preset selector:
  ```
  [Preset Selector ▼]  [💾 Save]  [🗑 Delete]
  ```
- Botão **Save**: captura params atuais do form, pede nome via `prompt()`, chama `invoke('create_preset', {preset})`
- Botão **Delete**: confirma com `confirm()`, chama `invoke('delete_preset', {id})`
- Recarregar lista de presets após save/delete

### 2.3 Persistência
- Já implementado: `create_preset` / `delete_preset` salvam em `~/.config/mediaforge/presets.json`
- Nada a mudar no Rust

**Arquivo:** `main.js`
- Para cada job com status `done`/`failed`/`cancelled`, adicionar botão `↺` que:

  - Status `pending`
  - Adiciona ao final da fila

- Já existe `btn-clear-files` para arquivos
- Adicionar `btn-clear-queue` no footer, ao lado de "Queue"
- Remove todos os jobs com status `done`/`failed`/`cancelled` (mantém `pending`/`running`)

## 5. Splash Screen

### 5.1 Resolução ideal
- Janela: 1400×900
- Splash deve ser centralizado, max-height: 50% da viewport
- Tamanho recomendado do PNG: **800×400** (boa qualidade em HiDPI sem ficar enorme)
- Usar `max-w-xs` (320px) como fallback, `max-h-40` (160px)
- Ideal: `max-w-[400px] max-h-[200px]` com `object-contain`

### 5.2 CSS do splash
- Fundo: `bg-[#0b0b12]` (igual ao app)
- Logo centralizado com fade-out de 500ms
- Versão no canto superior direito já existe
